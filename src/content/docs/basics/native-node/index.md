# 原生节点管理

原生节点是 TKE 面向 Kubernetes 场景提供的增强型节点形态。与普通节点相比，原生节点支持声明式基础设施管理、节点故障自愈、专用调度能力、资源利用率优化和更丰富的节点参数配置。

## 与标准节点池的区别

| 维度 | 标准节点池 | 原生节点 |
|------|------------|----------|
| 管理对象 | TKE 节点池 + AS 组 + CVM | `MachineSet` / `Machine` |
| 管理方式 | 云 API / 控制台 | Kubernetes API / 云 API / 控制台 |
| 伸缩方式 | 依赖 AS 和节点池配置 | 原生节点自研伸缩能力 |
| 调度能力 | Kubernetes 默认调度能力为主 | 支持原生节点专用调度能力 |
| 适用场景 | 通用生产节点池 | 高利用率、声明式运维、FinOps 优化 |

!!! note "目录边界"
    标准节点池文档放在 `node-pool/`；原生节点文档放在 `native-node/`；超级节点文档放在 `supernode/`。三者都是节点管理能力，但底层模型和操作方式不同。

## 使用建议

1. 新建原生节点时优先通过 `MachineSet` 管理，不要零散创建单个节点。
2. 删除原生节点前先降低 `MachineSet` 副本数，否则控制器可能自动补回节点。
3. 原生节点的自动伸缩由 TKE 原生能力管理，与普通节点池 AS 伸缩策略不同。
4. 对生产业务先使用标签或污点灰度引入原生节点，再逐步迁移核心工作负载。
