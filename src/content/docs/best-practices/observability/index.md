# 可观测性

可观测性最佳实践关注生产环境的指标、日志、链路追踪、告警和成本治理。具体开启和查看方式请参考 [运维监控](../../observability/)。

## 实践清单

| 主题 | 重点 |
|------|------|
| [监控告警](monitoring.md) | 指标采集、PromQL、告警规则和看板 |
| [日志采集](logging.md) | 日志结构、索引、留存和检索 |
| [链路追踪](tracing.md) | trace_id、服务拓扑、跨服务延迟 |

## 生产设计原则

1. 指标用于发现异常，日志用于解释原因，事件和审计用于还原时间线。
2. 告警必须可行动，避免把所有指标波动都推送给值班人。
3. 日志字段和 trace_id 在应用设计阶段确定，避免上线后补采集。
4. 观测数据有成本，保留周期、索引字段和采样率需要按业务等级设计。
5. 每个核心服务至少具备 Golden Signals：流量、错误率、延迟和饱和度。
