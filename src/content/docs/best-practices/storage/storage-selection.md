# 存储选型

存储选型先看访问模式，再看性能、数据安全和成本。

| 诉求 | 推荐 |
|------|------|
| 单 Pod 独占读写 | CBS |
| StatefulSet 每副本独立数据盘 | CBS + volumeClaimTemplates |
| 多 Pod 共享文件 | CFS |
| 大文件、数据集、备份归档 | COS |
| 临时缓存 | emptyDir、本地盘、本地缓存 |

## 设计原则

1. 数据库主数据目录优先使用 CBS，不用 COS 挂载替代块存储。
2. 多副本共享读写优先使用 CFS，并评估并发写一致性。
3. 大文件分发、模型和备份优先使用 COS 或 CFS/COS 组合。
4. 核心数据使用 `Retain` 回收策略，并配置快照或外部备份。
5. 所有 PVC 都加上业务、环境、责任人和成本标签。
